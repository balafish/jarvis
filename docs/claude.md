# Claude 協作指南

> 此文件記錄與 AI 助手（Claude）協作的最佳實踐與重要原則

---

## 互動模式

### 省錢模式（討論階段）

當進入討論階段時，啟動「省錢模式」：

- **簡潔回覆**：不需完整句子，重點列表即可
- **確認優先**：先確認理解，再行動
- **不主動產出**：討論時不執行文件撰寫
- **編號選項**：提供選項時用數字，方便快速回覆

**啟動方式**：說「省錢模式啟動」

### 產出模式（執行階段）

當需要實際產出時：

- **完整內容**：不精簡，完整撰寫
- **優化語句**：適合對外發表的品質
- **多語言**：依需求產出中/英/越版本

**啟動方式**：說「開始產出」或「產出模式」

## 記憶與持續性

### 問題

Claude 不會跨對話保留記憶，關閉視窗後資訊會遺失。

### 解決方案

使用 `discussion-notes.md` 作為持久化記憶：

1. **離開前**：自動更新筆記，記錄進度與待辦事項
2. **推送 Git**：確保記錄不會遺失
3. **回來時**：讀取筆記，快速接軌

### 觸發條件

當用戶說「我要離開」時，自動執行：

1. 更新 `discussion-notes.md`
2. 記錄：
   - 目前進度
   - 已確認事項
   - 待辦事項
   - 下次繼續的方向
3. Git commit & push

## 協作原則

### 先討論，後執行

```
討論 → 確認 → 產出 → 審核 → 推送
```

- 不要在討論階段就執行文件撰寫
- 等用戶確認方向後再開始產出

### 資訊收集

當資訊不足時：

- 列出需要的資訊清單
- 讓用戶逐項補充
- 不要猜測或假設

### 確認機制

重要決策前確認：

- 「正確？」
- 「是這樣嗎？」
- 「要調整嗎？」

## 效率技巧

### 快速回覆

- 用數字選擇選項：「1」「2」
- 簡短確認：「是」「對」「正確」
- 指令式回覆：「推送」「繼續」

### 批次處理

- 一次提供多項資訊
- 用條列式整理
- 減少來回次數

### 範例

```
用戶：1.Q1 2.Q2 3.持續 4.無
AI：收到。（整理後確認）
用戶：正確
AI：更新文件並推送。
```

## Git 工作流程

### 何時推送

- 完成一個階段性任務
- 用戶要離開時
- 重要資訊更新後

### Commit 訊息

- 簡潔明確
- 說明做了什麼
- 使用中文或英文皆可

## 常用指令

| 指令 | 效果 |
|------|------|
| 省錢模式啟動 | 進入精簡討論模式 |
| 開始產出 | 進入完整產出模式 |
| 推送 | Git add + commit + push |
| 我要離開 | 更新筆記 + 推送 + 摘要 |
| 繼續 | 接續上次進度 |

---

## 注意事項

1. **不要猜測**：資訊不足時詢問，不要假設
2. **確認理解**：重要事項先確認再執行
3. **保持簡潔**：討論時不需長篇大論
4. **及時儲存**：重要進度要推送到 Git
5. **尊重決策**：用戶說不改就不改
